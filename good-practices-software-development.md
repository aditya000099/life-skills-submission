# Good Practices for Software Development – Review

## Question 1

### Which points were new to you?

-I wasn't aware about the tools like TimeLimit and Freedom, which helps us to make our life organised.

-The idea of over-communicating. I used to think communicating only when needed is enough, but now I understand that in remote work it is better to inform the team even about small delays or issues.

-I learned that it’s better to use group chats instead of private messages so more people see updates.

-The correct way to ask questions when stuck. Like instead of saying "It's not working", we should explain the problem clearly, mention what we tried, and share logs or enough info to debug the error.

-I learned that turning on video during meetings improves rapport and communication with the team.
